Жүйе нақты уақыт режимінде бетті анықтап, экран алдындағы болу ұзақтығын есептейді және қорытынды аналитика ұсынады. Жобаның сипаттамасы Бұл жоба студенттердің күнделікті оқу жүктемесін жүйелеуге, тапсырмаларды уақытында орындауға және уақытты тиімді бөлуге көмектесетін веб-негізделген жоспарлау құралын құруды көздейді. Күтілетін нәтиже Қосымша студентке жеке оқу жоспарын құруға мүмкіндік беріп, оқу уақытының тиімділігін арттырады. Құралдар: Зерттеу барысында оқу жобалық тапсырмалар GitHub веб-сервисі негізінде ұйымдастырылады, ол студенттердің бірлесіп жұмыс жасауын және жобаның орындалу процесін бақылауды қамтамасыз ететін тегін платформа болып табылады. Қосымша көмекші құрал ретінде GenAI жүйелерін (ChatGPT, Copilot, Gemini) қолдануға рұқсат етіледі. 5. Интервенция және академиялық адалдық қағидалары Эксперимент барысында GenAI құралдары көмекші ресурс ретінде қарастырылады. Студенттерге GenAI-ді жобаны әзірлеудің кез келген кезеңінде қолдануға рұқсат беріледі, алайда дайын шешімдерді немесе толық кодты механикалық түрде көшіруге жол берілмейді. Әрбір GenAI қолдану жағдайы оның қолданылған кезеңі мен мақсатын көрсете отырып тіркеледі. Академиялық адалдық қағидалары эксперимент басталар алдында түсіндіріліп, студенттерден оларды сақтау жөнінде келісім алынады. 6. Процедура Эксперимент барысында студенттер 3–4 адамнан тұратын шағын топтарға біріктіріледі. Әр топта келесі рөлдер міндетті түрде бөлінеді: Жоба менеджері (Project Manager), Жоба архитекторы (Software Architect), Әзірлеуші (Developer). 1. Жоба архитекторы (Software Architect) Негізгі жауапкершілігі – бағдарламалық жүйенің жалпы архитектурасын және техникалық шешімдерін жобалау. Функциялары:  Бизнес-талаптарды техникалық мүмкіндіктермен сәйкестендіру.  Архитектуралық тәсілді таңдау: MVC, клиент-сервер, көп деңгейлі архитектура (layered architecture) және т.б.  Жоба құрылымын жобалау. Модульдер, пакеттер және компоненттердің өзара әрекеттесу жолдарын анықтау.  Жобаның мақсатына сай құралдарды бекіту.  API мен деректер форматтарын анықтау.  Жобаның бастапқы жоспардан ауытқымауын қадағалау. 2. Әзірлеуші (Developer) Негізгі жауапкершілігі – жобаның функционалдығын бекітілген архитектураға сәйкес бағдарламалық түрде іске асыру. Функциялары:  Тиімді және оқылатын код жазу.  Жобаның негізгі алгоритмдері мен функцияларын құрастыру.  Мәліметтер қорын жобалау, сұраныстар жазу және деректерді басқару.  Кодтың әрбір бөлігінің дұрыс жұмыс істейтінін тексеру.  Командаластарының кодын тексеру және сапасын қадағалау. 3. Жоба менеджері (Project Manager) Негізгі жауапкершілік – командалық жұмысты ұйымдастыру және үйлестіру. Функциялары:  Жобаны тапсырмаларға декомпозициялау (бөлшектеу);  Рөлдер мен жауапкершілікті бөлу;  Мерзімдерді жоспарлау (milestones – бақылау нүктелері, дедлайндар);  Тапсырмалардың орындалуын бақылау;  Команда ішіндегі коммуникация;  Тәуекелдерді басқару. Жобаны орындау кезінде топ мүшелері жобалық жоспарды бірлесіп әзірлейді, рөлдерді саналы түрде таңдайды және GenAI құралдарын қолдану немесе қолданбау туралы шешімді өздері қабылдайды. GenAI қолданылған жағдайда, оның қай кезеңде, қандай мақсатта және не үшін пайдаланылғаны арнайы тіркеу арқылы белгіленеді. Жобалық жұмыс бес апталық кезеңдік цикл бойынша ұйымдастырылады: 1-апта (Analysis): жобалық тапсырманың талаптарын талдау, мақсатты нақтылау және күтілетін нәтижелерді анықтау; 2-апта (Planning): шешім жоспарын құру, жобаның архитектурасын және негізгі модульдерін жобалау; 3-апта (Implementation): жобаны жүзеге асыру; 4-апта (Control): бағдарламалық қателерді анықтау, түзету және шешімнің тұрақтылығын тексеру; 5-апта (Defense): жобаны қорғау, орындалған шешімді негіздеу және рефлексия жүргізу. 7. Есепті шешу көрсеткіші (Problem-Solving Performance Indicator – PSPI) Студенттердің жобалық тапсырмаларды орындау сапасы Problem-Solving Performance Indicator (PSPI) арқылы бағаланады. PSPI жобаны әзірлеудің бес кезеңін қамтиды: Analysis, Planning, Implementation, Control және Defense. Әр кезең 0–20 балл аралығында бағаланып, жалпы PSPI көрсеткіші 0–100 балл шкаласында есептеледі. Бұл көрсеткіш есепті шешу сапасын кешенді түрде бағалауға мүмкіндік береді. 8. Деректерді жинау және талдау Зерттеу деректері бірнеше көзден жинақталады: GitHub commit тарихы, жобалық құжаттамалар, GenAI қолдану журналдары және жобаны қорғау нәтижелері. Алынған деректер сипаттамалық статистика әдістері арқылы өңделіп, PSPI көрсеткіштері мен GenAI қолдану сипаттамалары арасындағы байланыстар талданады. 9. Этика және деректерді басқару Зерттеу барысында этикалық қағидалар толық сақталады. Қатысушылардың жеке деректері жиналмайды, барлық нәтижелер анонимді түрде өңделеді. Жиналған деректер қауіпсіз ортада сақталып, тек ғылыми мақсатта пайдаланылады. 10. Шектеулер және валидтілік Зерттеудің ішкі валидтілігі бірдей жобалық тапсырмалар мен бағалау критерийлерін қолдану арқылы қамтамасыз етіледі. Сыртқы валидтілік зерттеудің нақты білім беру контекстімен шектелуі мүмкін, сондықтан нәтижелерді басқа оқу деңгейлеріне жалпылау шектеулі. from flask import Flask, render_template, request, redirect
import sqlite3
import os

app = Flask(__name__)

# базаға қосылу
def get_db():
    path = os.path.join(os.getcwd(), "database.db")
    conn = sqlite3.connect(path)
    conn.row_factory = sqlite3.Row
    return conn

# басты бет
@app.route("/")
def home():
    conn = get_db()
    tasks = conn.execute("SELECT * FROM tasks").fetchall()
    conn.close()
    return render_template("index.html", tasks=tasks)

# тапсырма қосу
@app.route("/add", methods=["GET", "POST"])
def add():
    if request.method == "POST":
        title = request.form["title"]
        deadline = request.form["deadline"]
        priority = request.form["priority"]

        conn = get_db()
        conn.execute(
            "INSERT INTO tasks (title, deadline, priority) VALUES (?, ?, ?)",
            (title, deadline, priority)
        )
        conn.commit()
        conn.close()

        return redirect("/")

    return render_template("add_task.html")

# өшіру
@app.route("/delete/<int:id>")
def delete(id):
    conn = get_db()
    conn.execute("DELETE FROM tasks WHERE id = ?", (id,))
    conn.commit()
    conn.close()
    return redirect("/")

if __name__ == "__main__":
    app.run(debug=True)
